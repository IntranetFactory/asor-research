## Overview

The `googleworkspace/cli` project implements the Agent Skills format to expose Google Workspace APIs as portable, composable agent skills. Its skills catalog ‚Äî auto-generated by `gws generate-skills` ‚Äî organizes 97+ skills across four tiers: Services (raw API wrappers), Helpers (single-operation shortcuts), Personas (role-based bundles), and Recipes (multi-step workflows). This taxonomy is a concrete reference for how to structure a large skill library.

## Four-Tier Taxonomy

The catalog's four categories map to increasing levels of abstraction and composability:

| Tier | Count | Purpose |
|------|-------|---------|
| **Services** | 19 | Core API wrappers; one skill per Google Workspace service |
| **Helpers** | 25 | Single-operation shortcuts scoped to one service |
| **Personas** | 10 | Role-based bundles aggregating multiple services |
| **Recipes** | 41 | Multi-step cross-service workflows with real commands |

### Services

Each Service skill wraps a single Google Workspace API and is named `gws-<service>`:

- `gws-drive`, `gws-sheets`, `gws-gmail`, `gws-calendar`, `gws-docs`, `gws-slides`
- `gws-tasks`, `gws-people`, `gws-chat`, `gws-meet`, `gws-forms`, `gws-keep`
- `gws-classroom`, `gws-events`, `gws-script`, `gws-workflow`
- `gws-admin-reports` (audit logs, usage reports)
- `gws-modelarmor` (content safety filtering)
- `gws-shared` (authentication, global flags, output formatting shared across all skills)

### Helpers

Helpers follow `gws-<service>-<operation>` naming and target a single, common action:

- **Gmail:** `gws-gmail-send`, `gws-gmail-triage`, `gws-gmail-reply`, `gws-gmail-reply-all`, `gws-gmail-forward`, `gws-gmail-read`, `gws-gmail-watch`
- **Drive:** `gws-drive-upload`
- **Sheets:** `gws-sheets-append`, `gws-sheets-read`
- **Calendar:** `gws-calendar-insert`, `gws-calendar-agenda`
- **Docs:** `gws-docs-write`
- **Chat:** `gws-chat-send`
- **Events:** `gws-events-subscribe`, `gws-events-renew`
- **Model Armor:** `gws-modelarmor-sanitize-prompt`, `gws-modelarmor-sanitize-response`, `gws-modelarmor-create-template`
- **Workflow:** `gws-workflow-standup-report`, `gws-workflow-meeting-prep`, `gws-workflow-email-to-task`, `gws-workflow-weekly-digest`, `gws-workflow-file-announce`
- **Apps Script:** `gws-script-push`

### Personas

Personas bundle multiple service skills into role-specific agents. Named `persona-<role>`:

| Persona | Role Description |
|---------|----------------|
| `persona-exec-assistant` | Manage an executive's schedule, inbox, and communications |
| `persona-project-manager` | Coordinate projects ‚Äî track tasks, schedule meetings, and share docs |
| `persona-hr-coordinator` | Handle HR workflows ‚Äî onboarding, announcements, and employee comms |
| `persona-sales-ops` | Manage sales workflows ‚Äî track deals, schedule calls, client comms |
| `persona-it-admin` | Administer IT ‚Äî monitor security and configure Workspace |
| `persona-content-creator` | Create, organize, and distribute content across Workspace |
| `persona-customer-support` | Manage customer support ‚Äî track tickets, respond, escalate issues |
| `persona-event-coordinator` | Plan and manage events ‚Äî scheduling, invitations, and logistics |
| `persona-team-lead` | Lead a team ‚Äî run standups, coordinate tasks, and communicate |
| `persona-researcher` | Organize research ‚Äî manage references, notes, and collaboration |

The `persona-hr-coordinator` is directly relevant to blended workforce tooling: it addresses onboarding, internal announcements, and employee communication ‚Äî the same workflow surface that Workday ASOR targets for AI agents embedded in HR processes.

### Recipes

Recipes are multi-step, cross-service workflows packaged as single skills. Named `recipe-<task>`. Selected examples grouped by theme:

**Email management**
- `recipe-label-and-archive-emails` ‚Äî Apply Gmail labels and archive matching messages
- `recipe-draft-email-from-doc` ‚Äî Read a Google Doc and compose it as an email body
- `recipe-save-email-attachments` ‚Äî Save Gmail attachments to Drive
- `recipe-forward-labeled-emails` ‚Äî Forward labeled messages to another address
- `recipe-create-vacation-responder` ‚Äî Enable Gmail out-of-office reply with date range

**Document & Drive workflows**
- `recipe-create-doc-from-template` ‚Äî Copy a Docs template, fill content, share
- `recipe-organize-drive-folder` ‚Äî Create folder structure and sort files
- `recipe-share-folder-with-team` ‚Äî Share Drive folder with collaborators
- `recipe-backup-sheet-as-csv` ‚Äî Export Sheets to CSV for local backup
- `recipe-generate-report-from-sheet` ‚Äî Read Sheet data and write a formatted Docs report

**Calendar operations**
- `recipe-schedule-recurring-event` ‚Äî Create recurring calendar events with attendees
- `recipe-find-free-time` ‚Äî Query free/busy status across users to find a slot
- `recipe-block-focus-time` ‚Äî Create recurring focus blocks
- `recipe-create-events-from-sheet` ‚Äî Read event data from Sheets and bulk-create calendar entries

**Cross-service automation**
- `recipe-post-mortem-setup` ‚Äî Create a Docs post-mortem, schedule a Calendar review, notify via Chat
- `recipe-send-team-announcement` ‚Äî Post announcement via both Gmail and Chat
- `recipe-sync-contacts-to-sheet` ‚Äî Export Google Contacts to a Sheets spreadsheet
- `recipe-share-event-materials` ‚Äî Share Drive files with all Calendar event attendees

## Naming Conventions

The catalog demonstrates a consistent prefix-based namespace strategy:
- `gws-` ‚Üí all core and helper skills (prevents collision with non-Workspace skills)
- `persona-` ‚Üí role bundles
- `recipe-` ‚Üí multi-step workflows

This prefix pattern is not mandated by the Agent Skills spec but is a best practice for large catalogs.

## Auto-Generation

The index file is generated by `gws generate-skills`, not hand-authored. This implies the toolchain can introspect skill directories and produce an up-to-date catalog automatically ‚Äî a useful pattern for maintaining discoverability as a library scales.
